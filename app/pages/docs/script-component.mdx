title: Componente <Script>
sidebar_label: <Script>

El componente Script de Blitz.js le permite a los desarrolladores
seleccionar la prioridad de la carga de scripts de terceros. Esto tiene el
fin de ahorrarle tiempo al desarrollador y mejorar el rendimiento.

Los sitios web suelen necesitar scripts de terceros para análisis (o
_analytics_), anuncios, complementos de soporte al cliente, y gestión del
consentimiento. Sin embargo, estos scripts tienden a ser pesados y suelen
ralentizar al sitio y perjudicar la experiencia del usuario. Los
desarrolladores normalmente se esfuerzan al decidir dónde los ubican en la
aplicación para lograr el menor tiempo de carga posible.

Con `<Script>` puedes decidir la estrategia a utilizar con `strategy` y
Blitz.js optimizará el script:

- `beforeInteractive`: Para scripts críticos que necesitan ser descargados
  y ejecutados **antes** de que la página sea interactiva (como detectores
  de bots o manejadores de consentimiento). Estos scripts son inyectados
  en el HTML inicial desde el servidor y son ejecutados antes que el resto
  de JavaScript.
- `afterInteractive` (**por defecto**): Para scrips que pueden ser
  descargados y ejecutados **después** de que la página sea interactiva
  (como manejadores de etiquetas o _analytics_). Estos scripts son
  inyectados en el lado del cliente y se ejecutarán luego de la
  hidratación.
- `lazyOnload`: Para scripts que puede esperar a que la página esté
  totalmente cargada (como chats de soporte o componentes de redes
  sociales).

> **Nota:** estas estrategias de carga funcionan igualmente para scripts
> de una sola línea envueltos con `<Script>`. Mira el ejemplo más abajo.

## Uso {#usage}

Anteriormente, necesitabas definir las etiquetas `script` dentro del
componente `Head` de tu página Blitz.js.

```js
// Antes

// pages/index.js
import { Head } from "blitz"

function Home() {
  return (
    <>
      <Head>
        <script
          async
          src="https://www.google-analytics.com/analytics.js"
        />
      </Head>
    </>
  )
}
```

Con `<Script>`, ya no necesitas envolverlos en `<Head>`. Asimismo,
`<Script>` **no** debe ser usado dentro de `pages/_document.js` ya que
`<Script>` tiene funcionalidad en el lado del cliente para asegurar el
orden de carga. Por ejemplo:

```js
// Ahora

// pages/index.js
import { Script } from "blitz"

function Home() {
  return (
    <>
      <Script src="https://www.google-analytics.com/analytics.js" />
    </>
  )
}
```

## Ejemplor {#examples}

### Carga de polyfills {#loading-polyfills}

```js
import { Script } from "blitz"
;<Script
  src="https://polyfill.io/v3/polyfill.min.js?features=IntersectionObserverEntry%2CIntersectionObserver"
  strategy="beforeInteractive"
/>
```

### Lazy-loading {#lazy-loading}

```js
import { Script } from "blitz"
;<Script
  src="https://connect.facebook.net/en_US/sdk.js"
  strategy="lazyOnload"
/>
```

### Ejecución de código luego de cargar (`onLoad`) {#after-load}

```js
import { Script } from "blitz"
;<Script
  id="stripe-js"
  src="https://js.stripe.com/v3/"
  onLoad={() => {
    this.setState({ stripe: window.Stripe("pk_test_12345") })
  }}
/>
```

### Scripts de una sola línea {#inline}

```js
import {Script} from 'blitz'

<Script strategy="lazyOnload">
  {`document.getElementById('banner').removeClass('hidden')`}
</Script>

// or

<Script
  dangerouslySetInnerHTML={{
    __html: `document.getElementById('banner').removeClass('hidden')`
  }}
/>
```

### Pasar atributos al script {#forwarding-attributes}

```js
import { Script } from "blitz"
;<Script
  src="https://www.google-analytics.com/analytics.js"
  id="analytics"
  nonce="XUENAJFW"
  data-test="analytics"
/>
```
